Use this guide to add a new resource-authored dungeon encounter that can appear on generated Encounter nodes.

## Prerequisites

- Encounter files live under `res://core/dungeon/encounters/events/`.
- `res://core/dungeon/encounters/default_dungeon_encounter_pool.tres` scans encounter folders and provides the default scene.
- Encounter nodes are generated by `res://core/dungeon/dungeon_floor_generator.gd`.
- Encounter scenes must implement `setup(encounter_data, context)` and emit `encounter_finished(result: Dictionary)`.
- The default reusable scene is `res://scenes/dungeon/encounters/DungeonChoiceEncounter.tscn`.

## Add the encounter data

1. Create a new `DungeonEncounterData` resource under `res://core/dungeon/encounters/events/`.
2. Use a lowercase file name that matches the encounter ID, such as `sealed_chest.tres`.
3. Set:
   - `id`: stable `StringName`, such as `&"sealed_chest"`.
   - `display_name`: player-facing title.
   - `description`: text shown by the default choice scene.
   - `valid_floor_layers`: floor numbers where this encounter can appear. Leave empty only when it can appear on any floor.
   - `weight`: weighted random chance within the valid floor pool.
   - `scene_override`: leave empty for the default choice scene, or assign a custom encounter scene.
   - `choices`: one or more inline choice dictionaries.
4. Confirm the file is under a folder listed in `DungeonEncounterPool.scan_roots`.

## Add choices and effects

Each choice is a dictionary in `choices`.

```gdscript
{
	"label": "Open the chest",
	"description": "The lock is warm to the touch.",
	"effects": [
		{"id": &"damage", "amount": 5},
		{"id": &"stat", "stat": "DEX", "amount": 2, "duration": 90.0},
	]
}
```

For damage, use:

```gdscript
{"id": &"damage", "amount": 8}
```

For a temporary stat modifier, use:

```gdscript
{"id": &"stat", "stat": "STR", "amount": 2, "duration": 120.0}
```

For a permanent stat modifier, use:

```gdscript
{"id": &"stat", "stat": "INT", "amount": 1, "permanent": true}
```

Damage subtracts from persistent run HP. Temporary stat modifiers expire as the run timer advances; permanent modifiers last until the run ends.

## Use a custom scene

Use a custom scene when the encounter needs special presentation or a flow beyond the default text-and-choice panel.

1. Create a scene under `res://scenes/dungeon/encounters/`.
2. Add a script with a `##` header comment.
3. Implement:

```gdscript
signal encounter_finished(result: Dictionary)

func setup(encounter_data: Resource, context: Dictionary) -> void:
	pass
```

4. Emit a complete result when the encounter is done:

```gdscript
encounter_finished.emit({
	"mode": "complete",
	"choice_index": 0,
})
```

5. Assign the custom scene to the encounter resource's `scene` field.

Only `mode = "complete"` is handled right now. Combat, route, teleport, or floor-change modes are reserved for later.

## Validate behavior

- Start two runs with the same seed and confirm the same Encounter nodes use the same `encounter_id`s.
- Start a run with a different seed and confirm the layout or encounter selection can differ.
- Click an Encounter node and confirm the scene loads over the dungeon map.
- Select each choice and confirm the node completes after `encounter_finished`.
- Confirm damage changes persistent HP in the global HUD and the next fight.
- Confirm temporary stat modifiers expire as run time advances.
- Run the headless load command after changing scripts/resources:

```powershell
& 'H:\Apps\Godot\Godot_v4.6.2-stable_win64_console.exe' --headless --path . --quit
```
